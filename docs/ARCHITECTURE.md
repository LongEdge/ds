## 1. 系统概述

DS（Distributed Services）是一个**分布式 AI 视觉模型推理平台**，采用 Master-Node 架构，支持多任务调度、节点管理、模型热加载、存储抽象等企业级功能。

### 1.1 设计目标

- **分布式计算** — 支持多节点横向扩展，节点无状态
- **多模型支持** — 图像分类、目标检测、人脸关键点、皱纹分割、视频跟踪等
- **存储抽象** — 支持 MinIO、阿里云 OSS、Azure Blob、本地文件系统
- **任务可靠** — 防重复提交、失败重试（最多4次）、超时控制
- **实时监控** — 节点心跳、任务进度实时上报

### 1.2 技术栈

| 层级 | 技术 |
|-----|------|
| Master 服务 | Python 3 + Flask + Gunicorn |
| Node 节点 | Python 3 + multiprocessing + threading |
| 数据库 | SQLite |
| 消息队列 | Kafka（封装未启用）+ MQTT（封装未启用） |
| 任务调度 | APScheduler BackgroundScheduler |
| 存储 | MinIO / OSS / Azure Blob / Local |
| AI 模型 | YOLOv8 / UNet / 自研模型 |

## 3. 核心组件

### 3.1 Master 服务

| 组件 | 职责 |
|-----|------|
| `app.py` | Flask 应用入口，配置加载，Blueprint 注册，启动调度器 |
| `scheduler.py` | 定时任务（节点监控），可选：任务队列分发 |
| `taskService` | 任务创建/领取/终止/查询 |
| `nodeService` | 节点注册/注销/能力管理 |
| `monitorService` | 接收节点心跳、进度上报 |
| `reportService` | 报告生成与查询 |
| `sql.py` | SQLite 操作封装，任务分发核心逻辑 |

### 3.2 Node 节点

| 组件 | 职责 |
|-----|------|
| `ProcessFramework.py` | 多进程+多线程框架，任务拉取/执行/上报/终止 |
| `run_task_entry()` | 子进程入口，模型加载，任务循环执行 |
| `Tool App` | 具体业务逻辑（目标检测/模型训练/标注） |
| `Storages` | 存储抽象层 |

## 4. 数据流

### 4.1 任务创建 → 执行 → 完成

```
1. 平台/HTTP → POST /create-task/
   └─ Master: 创建任务记录 (status=init)
               │
2. Node 定时 → POST /handle/ (node_no=xxx)
   └─ Master: 查询任务，更新状态为 running，返回任务参数
               │
3. Node: 执行任务
   ├─ 进度上报 → POST /report-node-processing-batch/ (每秒)
   ├─ 心跳 → POST /report-node-live-status/ (每5秒)
   └─ 整体进度 → POST /update-task-entire-progress/ (每秒)
               │
4. 完成 → status=ok/failed
```

### 4.2 节点拉取任务策略

```
Node 请求 handle(node_no)
    │
    ├─ Debug模式: 仅领取绑定到自己的任务
    │
    └─ 发布模式:
       ├─ 优先级1: 之前失败/超时的自身任务 (run_num < 4)
       ├─ 优先级2: 未指派的公共就绪任务
       └─ 无任务: 返回空
```

## 8. 高可用设计（当前缺失 & 建议）

### 8.1 Master 高可用

| 当前 | 建议 |
|-----|------|
| 单 Master | 主备切换 + VIP 漂移 |
| SQLite 单文件 | 换 MySQL/PostgreSQL |
| 无 API 限流 | 添加限流中间件 |

### 8.2 Node 高可用

| 当前 | 建议 |
|-----|------|
| 节点心跳依赖自身上报 | Master 主动探测 |
| 节点崩溃后任务丢失 | 任务超时后自动重分配 |
| 无节点健康评分 | 添加任务失败率/负载评分 |

### 8.3 任务高可用

| 当前 | 建议 |
|-----|------|
| 失败重试最多4次 | 可配置重试策略 |
| 任务无超时强制终止 | 添加超时阈值 |
| SQLite 并发写锁 | 换用 MySQL + 连接池 |

## 10. 性能调优

### 10.1 Master

| 参数 | 当前值 | 建议值 | 说明 |
|-----|-------|-------|------|
| Gunicorn workers | ? | CPU×2+1 | 并行处理能力 |
| SQLite WAL | 未启用 | 启用 | 提升并发读写 |
| Scheduler 轮询间隔 | 0.2s (已禁用) | 1-5s | 降低数据库压力 |

### 10.2 Node

| 参数 | 建议值 | 说明 |
|-----|-------|------|
| 子进程复用模型 | 已实现 | 避免重复加载 |
| 进度上报合并 | 已实现 (批量) | 减少 HTTP 请求 |
| Progress 队列大小 | 建议加限流 | 防止内存溢出 |

### 10.3 数据库

```sql
-- 定期清理任务日志（建议定时任务）
DELETE FROM task_log_info WHERE deal_time < (SELECT MAX(deal_time) FROM Query) - 7*24*3600;

-- 添加索引
CREATE INDEX IF NOT EXISTS idx_query_status ON Query(deal_status);
CREATE INDEX IF NOT EXISTS idx_query_node ON Query(node_no);
CREATE INDEX IF NOT EXISTS idx_tasklog_report ON task_log_info(report_id);
```

## 11. 安全建议

| 风险 | 当前状态 | 建议措施 |
|-----|---------|---------|
| API 无认证 | ⚠️ 无任何认证 | 添加 API Key / JWT |
| 节点注册无验证 | ⚠️ 任意注册 | 白名单 IP |
| SQL 注入 | 🟢 PreparedStatement | 保持现状 |
| 敏感信息硬编码 | ⚠️ 配置中有 redact | 使用密钥管理服务 |
| RemoteLinux 明文密码 | ⚠️ 使用密码 | 改用 SSH Key |

## 12. 监控指标

| 指标 | 采集方式 | 告警阈值 |
|-----|---------|---------|
| Master 服务存活 | 外部探活 | 不可达 > 30s |
| 节点心跳超时 | `last_heartbeat` | > 30s 无心跳 |
| 任务卡住 | `start_time` | running > 30min |
| 任务失败率 | 统计 | > 10% |
| SQLite 文件大小 | 文件系统监控 | > 1GB |
| 队列饱和度 | `model_type_queue` | maxsize=1 恒满 |
