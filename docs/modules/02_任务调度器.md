## 🏗️ 核心数据结构

### model_type_queue_dict
```python
self.model_type_queue_dict = {
    "face_landmark": Queue(maxsize=1): [report_id, ...],
    "slim_wrinkle": Queue(maxsize=1): [report_id, ...],
    "bold_wrinkle": Queue(maxsize=1): [report_id, ...]
}
```
- **每种模型类型**维护一个**最大长度为 1 的队列**
- 队列满时，新任务**无法入队**（阻塞等待）

### select_reportid_dicts
```python
self.select_reportid_dicts = {
    "report_id1": {"task_type": "face_landmark"},
    "report_id2": {"task_type": "slim_wrinkle"}
}
```
- 记录已选择的 report_id 及其任务类型
- 避免重复分发

### `auto_prepare_task()` ⚠️ 已禁用
**功能**：自动将 `init` 状态任务分发到对应模型类型队列

```python
def auto_prepare_task(self):
    for task_type in self.model_type_list:
        if self.model_type_queue_dict[task_type].full():
            continue  # 队列满则跳过
        
        # 从数据库选择 init 状态的任务
        report_id = self.sql_monitor.auto_select_init_task(task_type)
        if report_id is None:
            continue
        
        # 查询任务候选节点
        task_info = self.sql_monitor.query_task_info_by_report_id(report_id)
        candidate_nodes = task_info['candidate_nodes']
        
        # 入队
        queue_data = {"report_id": report_id, "candidate_nodes": candidate_nodes}
        self.model_type_queue_dict[task_type].put(queue_data)
```

### `auto_appoint_task()` ⚠️ 已禁用
**功能**：从队列中取出任务，指派给空闲节点

```python
def auto_appoint_task(self):
    for _, type_queue in self.model_type_queue_dict.items():
        if type_queue.empty():
            continue
        
        queue_data = type_queue.queue[0]  # 不弹出，只查看
        candidate_nodes = queue_data['candidate_nodes']
        
        # 查询所有空闲节点 (status=2)
        free_nodes = self.sql_monitor.query_node_by_status(2)
        
        # 策略选择：第一个既空闲又在候选列表中的节点
        select_free_node = None
        for free_node in free_nodes:
            if free_node in candidate_nodes:
                select_free_node = free_node
                break
        
        if select_free_node:
            # 1. 指派任务到节点
            self.sql_monitor.appoint_task(report_id, select_free_node)
            # 2. 更新任务状态 is_appointment = 1
            self.sql_monitor.update_appoint_task_status(report_id, 1)
            # 3. 弹出队列
            type_queue.get()
```

## 🔒 线程安全

```python
self.lock = threading.Lock()  # 保护 model_type_list 和 model_type_queue_dict
```

- 所有对 `model_type_queue_dict` 的读写操作都在 `with self.lock:` 内

## 📊 任务分发流程（设计但未启用）

```
任务创建 (init)
    ↓
auto_prepare_task() [0.2s 轮询]
    ↓ (入队到 model_type_queue_dict)
队列等待 (maxsize=1)
    ↓
auto_appoint_task() [0.2s 轮询]
    ↓ (节点指派 + 状态更新)
节点领取 (running)
    ↓
任务执行...
    ↓
结果上报 (ok/failed)
```

## ⚠️ 设计问题 & 风险

| 问题 | 严重程度 | 说明 |
|-----|---------|------|
| 队列 maxsize=1 | 🔴 高 | 任务处理速度依赖节点，队列易饱和 |
| 自动分发已禁用 | 🔴 高 | 实际靠节点主动拉取，非 Master 推送 |
| 无超时重试机制 | 🟡 中 | 任务卡住没有超时强制终止 |
| 候选节点选择策略简单 | 🟡 中 | 只选第一个匹配的节点，无负载均衡 |
| 调度器 0.2s 轮询 | 🟡 中 | 高频轮询，增加数据库压力 |

---

## 🎯 改进建议

1. **队列大小可配置化** — `MAXSIZE_QUEUE = 1` 应从 config.yml 读取
2. **启用自动分发** — 如果需要 Master push 模式，启用注释的 job
3. **添加任务超时** — 超过一定时间的 running 任务强制失败
4. **节点选择策略** — 实现负载均衡（最少任务节点优先）
5. **批量任务准备** — 一次从数据库取多条任务减少查询次数
