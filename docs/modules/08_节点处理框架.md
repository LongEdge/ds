## 📋 概述

| 属性 | 值 |
|-----|-----|
| 文件 | `ai-node-svr/code/src/ProcessFramework.py` |
| 类 | `ProcessorFramework` |
| 架构 | 多进程 (multiprocessing) + 多线程 (threading) |
| 用途 | Node 节点主框架，负责任务拉取、执行、进度上报 |

---

## 🏗️ 整体架构

```
主进程 (Main Process)
    │
    ├─ 信号处理 (SIGINT/SIGTERM)
    │
    └─ 启动 7 个守护线程
           │
           ├─ _read_progress_queue_thread    # 读取子进程进度
           ├─ _read_entire_queue_thread      # 读取整体进度回调
           ├─ node_heart_beat                # 心跳上报 (每5秒)
           ├─ node_task_processing            # 批量进度上报 (每秒)
           ├─ node_task_entire_processing    # 整体进度上报 (每秒)
           ├─ monitor_kill_signal            # 终止信号监控 (每1.5秒)
           └─ node_process_worker             # 任务拉取主循环 (每2秒)
                  │
                  └─ 子进程 (run_task_entry)
                         │
                         ├─ 加载 AI 模型 (一次)
                         └─ while True: 处理任务
```

## 🔧 核心组件

### 1. 多进程模型

```python
# 子进程入口
def run_task_entry(node_cfg, task_queue, progress_queue, entire_queue):
    # 1. 一次性加载模型
    tool_class = loadPyFile('App', 'CApp', 'src.tools.{}'.format(node_cfg['tool_name']))
    tool_ins = tool_class(node_cfg, dummy_callback)
    
    # 2. 循环处理任务
    while True:
        param = task_queue.get()
        if param is None: break
        
        try:
            tool_ins.ProcessTask(param)
            dummy_callback(100, "TASK_FINISHED_okESSFULLY", "ok")
        except Exception as e:
            dummy_callback(-1, f"TASK_ERROR: {e}", "failed")
```

**关键设计**：
- 模型只加载一次，任务队列式获取（**进程复用模型**）
- multiprocessing 使用 `spawn` 模式（**兼容 CUDA**）
- `task_queue` 控制任务分发
- `progress_queue` 传递进度回调
- `entire_queue` 传递整体进度回调

### 2. 状态管理

```python
self.live_status = 2       # 2=空闲, 1=繁忙
self.report_id = None      # 当前任务ID
self.db_id = -1            # 数据库ID
self.kill_flag = False      # 终止标志
self.current_task_status = None  # 'ok' / 'failed' / 'killed'
```

### 3. 子进程管理

```python
def ensure_child_process(self):
    if self.current_task_process is None or not self.current_task_process.is_alive():
        # 重燃子进程
        self.current_task_process = multiprocessing.Process(
            target=run_task_entry,
            args=(self.node_cfg, self._task_queue, self._progress_queue, self._entire_queue)
        )
        self.current_task_process.start()
```

## 🔄 主工作循环 (node_process_worker)

```python
def node_process_worker(self):
    while True:
        self.live_status = 2  # 空闲
        time.sleep(2)
        if self.report_id: continue  # 有任务时跳过拉取
        
        try:
            # 1. 向 Master 请求任务
            url = f"http://{master_ip}:{master_port}/ai-master-svr/handle/"
            res = requests.post(url, data={'node_no': node_id}, timeout=5)
            data = res.json()
            
            if not data.get('report_id'): continue
            
            # 2. 解析任务参数
            self.report_id = data['report_id']
            self.db_id = data.get('db_id', -1)
            self.current_param = json.loads(data.get('param', '{}'))
            
            # 3. 启动子进程执行
            self.ensure_child_process()
            self.live_status = 1  # 繁忙
            self._task_queue.put(self.current_param)
            
            # 4. 等待任务完成
            while self.report_id is not None:
                if not self.current_task_process.is_alive():
                    if not self.kill_flag and self.current_task_status is None:
                        # 子进程崩溃
                        self.current_task_status = "failed"
                        self.update_task_status("failed")
                    self.reset_all()
                    break
                time.sleep(1)
        except: continue
```

## ⚠️ 设计问题 & 风险

| 问题 | 严重程度 | 说明 |
|-----|---------|------|
| 子进程崩溃后无自动重启 | 🔴 高 | `ensure_child_process` 只在 `report_id` 为空时触发重启 |
| 任务执行结果依赖进程正常退出 | 🟡 中 | 子进程异常退出时可能丢失任务状态 |
| 无超时机制 | 🟡 中 | 任务可能永远卡住 |
| HTTP 超时 5s 可能丢任务 | 🟡 中 | 请求 Master 失败不重试 |
| 进度队列无界 | 🟡 中 | 内存可能被打满 |

---

## 🎯 改进建议

1. **添加任务超时** — 每个任务设置最大执行时间，超时强制终止
2. **子进程崩溃自动重启** — 无论是否有任务在执行
3. **请求重试机制** — Master 请求失败时指数退避重试
4. **Progress 队列限流** — 超出一定大小丢弃或压缩
5. **优雅退出** — 支持 Graceful Shutdown，等当前任务完成再退出
