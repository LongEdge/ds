| `vedio-split` | 视频抽帧 + 分类 + 检测 | 提取有效帧，保存图片和标注 |

## 🔧 核心流程

### vedio-split / vedio-imgrecogintion 流程

```
输入: 视频URL + MinIO输出配置
│
├─ 1. 初始化模型
│   ├─ cls_model = YOLO('yolov8x-cls.pt')   # 图像分类
│   └─ det_model = YOLO('yolov8n.pt')        # 目标检测
│
├─ 2. 逐帧读取视频
│   └─ 每隔 120 帧处理一次 (frame_count % 120 == 0)
│
├─ 3. 帧相似度过滤
│   └─ SSIM (结构相似度) < 0.8 才处理，否则跳过
│
├─ 4. 图像分类
│   └─ YOLO 分类模型预测类别
│
├─ 5. 目标检测
│   └─ YOLO 检测模型输出边界框
│
└─ 6. 结果存储
    ├─ 本地保存: out_path/images/{class_name}/*.jpg
    ├─ 本地保存: out_path/labels/{class_name}/*.json
    └─ 上传 MinIO: {bucket}/{prefix}/group-xxx/{class_name}_*.jpg
```

### 帧相似度过滤算法
```python
def calculate_frame_similarity(self, frame1, frame2):
    gray1 = cv2.cvtColor(frame1, cv2.COLOR_BGR2GRAY)
    gray2 = cv2.cvtColor(frame2, cv2.COLOR_BGR2GRAY)
    score, _ = ssim(gray1, gray2, full=True)
    return score  # SSIM < 0.8 表示差异足够大，保留该帧
```

## ⚠️ 问题 & 限制

| 问题 | 影响 |
|-----|------|
| 模型文件硬编码路径 | 部署时必须手动复制模型文件 |
| 无 GPU 利用率监控 | 无法确认是否真正用 GPU 推理 |
| 视频解码用 OpenCV | 某些编码格式可能不支持 |
| group-xxx 自动溢出 | 超过 1000 个文件自动创建新 group（逻辑可能有问题） |
