# 10 - 模型训练工具 (ds_tpsvr_0003_VisionModelTrain)

## 📋 概述

| 属性 | 值 |
|-----|-----|
| 目录 | `ai-node-svr/code/src/tools/ds_tpsvr_0003_VisionModelTrain/` |
| 主类 | `CApp` (封装 `CProcessor`) |
| 功能 | 视觉模型训练框架（UNet图像分割、人脸关键点、图像分类、视频跟踪） |

---

## 🏗️ 架构

```
CApp
    └─ CProcessor (Process.py)
            │
            ├─ 存储层 (conn/)
            │   ├─ Local.py      # 本地文件系统
            │   ├─ Minio.py     # MinIO 对象存储
            │   ├─ OSS.py       # 阿里云 OSS
            │   ├─ RemoteLinux.py # 远程 Linux 服务器
            │   └─ Storages.py  # 统一存储接口
            │
            ├─ 特征模块 (features/)
            │   ├─ faceLandmark/      # 人脸关键点检测
            │   ├─ imgClassify/        # 图像分类 (UNet)
            │   ├─ imgWrinkle/         # 皱纹分割 (UNet)
            │   └─ videoTracking/      # 视频跟踪
            │
            ├─ 处理流程 (process/)
            │   ├─ proc_comm.py        # 通用处理
            │   └─ imgBatchProcessTool.py # 批量图像处理
            │
            ├─ API 服务 (service/)
            │   ├─ api.py             # Flask API
            │   └─ task_manager.py    # 任务管理
            │
            └─ 工具 (util/)
                └─ log.py
```

## 🧠 模型模块

### 1. faceLandmark (人脸关键点)
```
trainFaceLandmark.py + module.py
- 输入: 人脸图片
- 输出: 68 点关键点坐标
- 用途: 人脸对齐、表情分析
```

### 2. imgClassify (图像分类)
```
module_unet.py + trainImgClassify.py
- 使用 UNet 架构进行图像分类/分割
- 支持多类别分类
```

### 3. imgWrinkle (皱纹分割)
```
src/features/imgWrinkle/v0_0_0/
├── module_unet.py
└── trainImgWrinkle.py
- 输入: 皮肤图片
- 输出: 皱纹区域掩码
- 皱纹类型: 泪沟纹、法令纹、综合皱纹
```

### 4. videoTracking (视频跟踪)
```
src/features/videoTracking/v0_0_0/
└── trainVideoTracking.py
- 输入: 视频帧序列
- 输出: 目标跟踪轨迹
```

## 🔌 API 服务 (service/api.py)

提供 HTTP API 接口供外部调用：

```python

POST /train/start      # 开始训练
POST /train/stop       # 停止训练
GET  /train/status     # 查询训练状态
GET  /train/logs       # 获取训练日志

# 模型管理
POST /model/save       # 保存模型
GET  /model/load       # 加载模型
```

## ⚠️ 当前问题

| 问题 | 风险 |
|-----|------|
| 训练代码与工具框架耦合 | 代码复用性差 |
| 无分布式训练支持 | 单机训练受限于 GPU 数量 |
| 无实验管理 | 训练结果难以追踪对比 |
| 模型版本管理缺失 | 无法回滚到旧版本 |
| 配置分散 | 多个 JSON/YAML 文件难以维护 |
