### 阿里云 OSS (OSS.py)

### Azure Blob (Azure.py)
```python
from azure.storage.blob import BlobServiceClient

client = BlobServiceClient.from_connection_string(conn_str)
container = client.get_container_client(container_name)

# 上传
with open(local_path, 'rb') as data:
    container.upload_blob(name=object_name, data=data)

# 下载
blob = container.get_blob_client(object_name)
with open(local_path, 'wb') as data:
    data.write(blob.download_blob().readall())
```

## 🔧 统一调度 (Storages.py)

```python
class CStorages:
    def uploadFile(self, conn_name, local_path, remote_path):
        conn_info = self.get_conn_info(conn_name)
        conn_type = conn_info['type']
        
        if conn_type == 'minio':
            return self.minio.upload(local_path, remote_path)
        elif conn_type == 'oss':
            return self.oss.upload(local_path, remote_path)
        elif conn_type == 'local':
            return self.local.copy(local_path, remote_path)
        elif conn_type == 'remote_linux':
            return self.remote.upload(local_path, remote_path)
```

## ⚠️ 问题 & 限制

| 问题 | 影响 |
|-----|------|
| 连接信息存储在平台 | 强耦合，无法独立运行 |
| 无连接池 | 每次请求都创建新连接（OSS/MinIO） |
| 无重试机制 | 网络抖动直接失败 |
| 无断点续传 | 大文件上传不稳定 |
| RemoteLinux 明文密码 | 安全风险 |

---

## 💡 改进建议

1. **引入连接池** — 复用连接，减少开销
2. **统一异常处理** — 定义存储异常基类
3. **断点续传** — 分片上传大文件
4. **异步上传** — 后台上传，不阻塞主流程
5. **安全加固** — 远程 Linux 使用 SSH Key 认证
